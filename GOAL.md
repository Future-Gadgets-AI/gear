# Goal — gear P2 exit ledger

## Mission

gear is a dark-factory content pipeline: canonical content specs and per-medium renderings move from issue to published post through blind review, with humans authoring and approving at the boundary — the line in between runs autonomously.

## P2 Exit Criteria

- [x] `plugin/contracts/` — canonical content rules (spec schema, voice contract, review rubric) — live
- [x] Per-medium renderer skills — live
- [x] `gear` plugin listed in cc-plugins (installable via url/https)
- [x] Site live, deploying automatically on merge to `main`
- [x] First post shipped end-to-end (issue → PR → blind review → human merge → published)
- [x] Series backlog seeded (draft issues queued for the next posts)

## Volatility

This file changes through a normal documentation pull request — no amendment ceremony required. Contrast `CONSTITUTION.md`, which changes only via its own amendment procedure and a version bump.
